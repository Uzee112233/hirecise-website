Security

Your data is safe with us

We take the security of your data and your candidates' data seriously — here's exactly what we do to protect it.

Last updated: June 2026

🔒

AES-256 Encryption

All data stored on our servers is encrypted at rest using AES-256, the same standard used by banks and governments.

🔐

TLS in Transit

All data transferred between your browser and our servers is encrypted using TLS 1.2 or higher.

🇪🇺

GDPR Compliant

We operate as a data processor under GDPR. Candidate data is handled lawfully and with appropriate safeguards.

🚫

Data Never Sold

We do not sell, rent, or broker your data or your candidates' data to any third party. Ever.

👤

Access Controls

Role-based access controls ensure only authorised team members can access candidate and account data.

🔍

Regular Reviews

We conduct regular security reviews of our infrastructure, dependencies, and access policies.

Infrastructure Security

Hirecise is hosted on enterprise-grade cloud infrastructure with:

  • Isolated environments for production, staging, and development
  • Automated backups with point-in-time recovery
  • DDoS protection at the network level
  • Uptime monitoring and automated alerting
  • Environment variables and secrets managed via encrypted vault — never hardcoded

Application Security

  • All user inputs are validated and sanitised server-side
  • Passwords are hashed using industry-standard algorithms (bcrypt)
  • Authentication uses short-lived JWT tokens with secure, HttpOnly cookies
  • Rate limiting is applied to all API endpoints to prevent abuse
  • CORS policies are enforced to restrict cross-origin access
  • Admin panel access is restricted by IP allowlist

Candidate Data Handling

Candidate CVs and application data are treated with the highest level of care:

  • CV files are stored in encrypted object storage with access restricted to the hiring organisation
  • AI processing is performed via a secure API call — data is not retained by the AI provider for training under our data processing agreement
  • Candidates are informed that their CV will be processed by AI at the time of submission
  • Data is retained only as long as necessary and deleted upon account closure or upon request

Payment Security

We do not store card details on our servers. All payment processing is handled by a PCI-DSS Level 1 compliant payment provider. We only store a tokenised reference to your subscription.

Responsible Disclosure

If you discover a security vulnerability in Hirecise, please report it responsibly to hello@hirecise.com with the subject line "Security Disclosure". We will acknowledge your report within 48 hours and work to resolve confirmed issues promptly. We ask that you do not publicly disclose vulnerabilities until we have had the opportunity to address them.

Questions

Have a security question or concern? Contact us at hello@hirecise.com. For general privacy questions, see our Privacy Policy.
